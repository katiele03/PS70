Week 7: Electronic Outputs

Assignment: Kinetic Sculpture – Ballerina Dancing to Beethoven’s Für Elise

For this week’s assignment, I created a different kinetic sculpture of a ballerina dancing to Beethoven's Für Elise. I first began by creating the ballerina, designing an outline, importing the outline into Fusion 360, and creating a sketch. I laser cut the DXF file and painted the ballerina.

Then, I 3D-printed a connector between the motor, which causes the ballerina to rotate, and the dowel, which the ballerina attaches to. Here is a slice of the connector.

Using the same code from last week’s kinetic sculpture assignment, I was able to get the motor to rotate continuously using an ESP32 microcontroller, a motor driver, and a motor. From the ESP32 to the motor driver, I connected GPIO18 to A1-A, GPIO19 to A1-B, 5 V to VCC, and GND to GND. I also connected the motor’s positive and negative wires to the motor driver.

After I got the motor to rotate, I began working on the speaker. I used an ESP32 microcontroller, a MAX98357A amplifier, and a small speaker. From the amplifier to the ESP32, I connected LRC → GPIO25, BCLK → GPIO5, DIN → GPIO26, VCC → 3.3 V, GND → GND, and GAIN → GND. The speaker’s positive and negative leads were connected to the amplifier terminals.

I could only find Beethoven's Für Elise in MP3 format, but I needed it as a .h file. I converted the MP3 to WAV using Audacity, changed the project rate to 48,000 Hz, set the track to Mono, and trimmed the clip to about 12 seconds.

I then converted the WAV file to a .h file so that it could be played through the ESP32 and MAX98357A amplifier. Below is part of the generated .h file.

I tested whether the speaker could play Für Elise using the test code modeled after I2S audio examples.

After hearing the music snippet play, I combined the two code snippets so that the ballerina would rotate to the music. Here is a short video of my ballerina dancing.
